# 需求与系统分析技能模板设计

## 1. 技能概述
- **名称**: requirement-system-analysis
- **描述**: 提供从原始需求到具体系统设计的完整分析框架，包含6个核心模块
- **适用范围**: 各种规模和类型的软件项目

## 2. 目录结构
```
.trae/skills/requirement-system-analysis/
├── SKILL.md              # 技能主文档
├── templates/            # 输出文档模板
│   ├── requirement_analysis.md.j2
│   ├── functional_design.md.j2
│   ├── nonfunctional_requirements.md.j2
│   ├── architecture_design.md.j2

│   └── implementation_roadmap.md.j2
└── examples/             # 示例案例
    └── e-commerce_platform_analysis.md
```

## 3. 核心模块设计

### 3.1 需求收集与分析模块
- **分析框架**: 5W2H法、用户故事映射、MoSCoW优先级排序
- **关键问题清单**: 业务目标、用户需求、约束条件、成功标准
- **评估标准**: 需求完整性、一致性、可测试性、可跟踪性
- **输出模板**: 需求规格说明书（SRS）

### 3.2 系统功能规划模块
- **分析框架**: 功能分解树、用例图、流程图
- **关键问题清单**: 核心功能、辅助功能、用户流程、功能依赖
- **评估标准**: 功能覆盖率、模块化程度、用户体验
- **输出模板**: 功能设计文档

### 3.3 非功能需求定义模块
- **分析框架**: FURPS+模型
- **关键问题清单**: 性能、可靠性、安全性、可扩展性、易用性
- **评估标准**: 可度量性、可实现性、与业务目标的一致性
- **输出模板**: 非功能需求文档

### 3.4 系统架构设计模块
- **分析框架**: C4模型、架构风格选择
- **关键问题清单**: 系统组件、组件间交互、数据流向、部署架构
- **评估标准**: 模块化、可扩展性、可维护性、安全性
- **输出模板**: 架构设计文档

### 3.5 技术选型评估模块
- **分析框架**: 技术雷达、SWOT分析
- **关键问题清单**: 语言框架、数据库、中间件、云服务
- **评估标准**: 成熟度、社区支持、性能、成本、团队熟悉度
- **输出模板**: 技术选型报告

### 3.6 项目实施路线规划模块
- **分析框架**: 敏捷开发、WBS分解
- **关键问题清单**: 里程碑、迭代计划、资源分配、风险评估
- **评估标准**: 可行性、合理性、灵活性
- **输出模板**: 项目实施路线图

## 4. 实现步骤
1. 创建技能目录结构
2. 编写SKILL.md主文档，包含所有模块的详细内容
3. 创建各模块的输出文档模板
4. 添加示例案例
5. 测试技能模板的完整性和可用性

## 5. 最佳实践
- 采用迭代式分析方法，逐步细化需求和设计
- 结合可视化工具（如Mermaid图表）增强文档可读性
- 邀请相关 stakeholders 参与评审，确保分析结果的准确性
- 保持文档的可维护性，定期更新以适应项目变化
